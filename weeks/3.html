깃 프로필 생성하기

프로필 만들기
프로필을 만드는 이유로는 누군가 내 깃허브에 접속을 했을 때 저장소만 반겨주기보다 내가 어떤사람이고 어떤기술을 사용할수있고 어떤작업물이 있고 등을 알려주어 나를 소개를 할 수 있기 때문입니다.

깃 프로필을 만드는 방법은 무척이나 쉽습니다.

바로 내 아이디와 동일한 repository를 생성하고 Readme.md를 생성하면 끝이 납니다.

Simple Icons에 접속하여 원하는 아이콘명과 색상정보를 받아옵니다.

- 뱃지레이블과 로고에는 아이콘 이름을 작성을 합니다.

- style에는
1. 엠보싱 효과 + 라운드 처리 : plastic
2. 라운드 처리 : flat
3. 네모박스 : flat-square
4. 네모박스 + 여백 : for-the-badge
5. 소셜 : social
5가지중에 한가지 선택하시면 됩니다.

- 배경색에는 색상정보를 입력합니다.

- 로고색상으로는 대부분 #ffffff를 사용하지만 원하는 색상이 존재 한다면 원하는 헥사코드를 가져와 사용하시면 됩니다.

프로필에서 pin을 이용하여 다양한 정보들을 보여줄 수 있는데요. 어떻게 사용하는지 알아보고자 합니다. 우선 깃토큰을 준비해주세요.
깃토큰 생성시 select scope에서 repo와 gist 체크는 꼭 해주셔야합니다.

1. 제목과 내용은 아무문자나 입력해 gist를 생성해주세요. gist를 생성을 하면 주소에 gist.github.com/gist번호가 나오는데 이때 나오는 gist번호를 잘 복사해둡니다.

2. 여기에 접속을 하여 Fork를 합니다.

3. Fork한 내 repo에서 setting에 접속한뒤 Security에 secrets and variables안에 들어가있는 actions를 선택합니다.

4. 카테고리를 선택하면 Environment secrets와 repository secrets가 있는데 repository secrets에서 new repository secret을 눌러 총 2가지의 secrets를 생성합니다.
    1. 제목 GH_TOKEN, 내용 토큰값
    2. 제목 GIST_ID, 내용 gist번호

5. repository의 actions에 들어가 enable을 눌러줍니다.

6. repository로 돌아가 workflows의 schedule.yml의 환경변수를 수정을 해주는데요

env:
    GH_TOKEN: ${{ secrets.GH_TOKEN }}
    GIST_ID:  ${{ secrets.GIST_ID }}
    TIMEZONE: Asia/Seoul

로 수정을 하고 저장을 해줍니다. 이때 문제가 없다면 프로필 페이지에 가서 customize your pins를 눌러 원하는 pin을 선택을 해주면 끝입니다.


다양한 pins를 정리한곳입니다. 필요한 pin이 있다면 사용하여 프로필을 예쁘게 꾸며봅시다.(클릭)

ERD

데이터 모델링과 ERD
데이터 모델링은 데이터를 분석하고 약속된 표기법으로 표현하는것을 말하는데요. 데이터 베이스 설계의 핵심 과정이기도 합니다. 그 이유는 SQL문의 성능을 효육적으로 작성할 수있게 만들기 때문입니다.

ERD란?

모델링을 할때 우리는 ERD라고 부르는 표기법을 사용하는데요.
Entity Relationship Diagram 의 약자로 흔히 ER다이어그램, ERD라고 부르는데요. 데이터들의 관계를 나타낸 도표입니다.

데이터 모델링작업을 하기 위해서는 알아야 하는 개념 4가지가 있습니다. 

  • 데이터의 수집/분석모델링을 하기전에 작업하며 보통 한 페이지에 필요한 데이터들을 수집해서 어떻게 데이터화 할 지 분석하는 단계입니다.
                  
  • 개념적 데이터 모델링데이터의 수집/분석을 마친 데이터들로 데이터들간의 관계를 구상하는게 목적입니다.
                  
  • 논리적 데이터 모델링개념적 데이터 모델링에 살을 덧대작업으로 개념적 데이터 모델링을 보고 key와 속성, 관계 등을 표시하면서 일관성을 확보하고 중복을 제거하여 데이터 구조를 얻는게 목적입니다.
논리적 데이터 모델링에서는 데이터를 추상적으로 작성하기 보다 구체적으로 데이터의 타입, 데이터간의 관계등을 함께 작성합니다.
                  
  • 물리적 데이터 모델링 최종적으로 데이터를 관리할 데이터베이스를 선택하고 선택한 데이터베이스에 실제로 테이블을 만드는 작업이며 SQL문을 작성하여 완성하는 단계입니다.
  1. 1. 데이터의 수집/분석을 통해 사용자의 요구를 정리하고 분석
  2. 2. 개념적 데이터 모델링에서 중요 개념들을 구분
  3. 3. 논리적 데이터 모델링에서 구분한 개념들을 구체화
  4. 4. 물리적 데이터 모델링에서 지금까지 정한 계획을 따라서 데이터베이스를 생성 
각 엔티티, 즉 테이블이 관계가 있는 경우에는 선을 이어서 관계를 맺어야 하는데요. 이때 실선으로 그으면 강한 관계를 나타내며 식별자 관계라고 부르고 점선으로 그으면 약한 관계를 나타내며 비식별자 관계라는 것을 알 수 있습니다.

간단하게 ERD 관계의 표현에 대해서 알아보고자 합니다. 4페이지에서 실선으로 그으면 강한 관계를 나타내며 식별자 관계라고 부르고 점선으로 그으면 약한 관계를 나타내며 비식별자 관계라고 설명을 드렸었는데요. 여기에 기호를 추가하여 관계를 표현해보려합니다.

각 테이블 별로 연결 하는데 두가지의 조건을 표시합니다.

1. 참여가 필수인가?

2. 어떤 대응을 하는가?
